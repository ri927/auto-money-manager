## Amplifyデプロイ

### 初回デプロイ手順

1. **AWS Amplifyコンソールにアクセス**
   - AWS Management Consoleにログイン
   - Amplifyサービスを開く

2. **アプリケーションの作成**
   - 「New app」→「Host web app」を選択
   - GitHubリポジトリを接続
   - ブランチを選択（mainブランチを推奨）
   - ビルド設定を確認（自動検出されます）

3. **環境変数の設定**（必要に応じて）
   - 「Environment variables」で環境変数を追加
   - 現時点では特に必要な環境変数はありません

4. **デプロイの実行**
   - 「Save and deploy」をクリック
   - デプロイが完了するまで待機（5〜10分程度）

5. **デプロイ完了の確認**
   - 「Deployment successfully completed」と表示されることを確認
   - 提供されたURLにアクセスしてアプリが動作することを確認

## Amazon Cognito設定

### 自己サインアップの無効化（重要）

**このアプリは個人使用を想定しており、外部ユーザーが自由にサインアップできないようにする必要があります。**

#### 手順

1. **AWS Cognitoコンソールにアクセス**
   ```
   AWS Management Console → Amazon Cognito
   ```

2. **User Poolを選択**
   - Amplifyが作成したUser Poolを選択
   - 名前は `amplify-` で始まる形式になっています

3. **Sign-up experienceの設定**
   ```
   左メニュー: Sign-up experience
   ```

4. **Self-service sign-upを無効化**
   - 「Self-service sign-up」セクションを見つける
   - **「Enable self-service sign-up」のチェックを外す**
   - 「Save changes」をクリック

5. **設定の確認**
   - 設定が保存されたことを確認
   - これにより、管理者（あなた）のみがユーザーを作成できるようになります

#### 設定完了後の効果

- ✅ フロントエンドの新規登録ページ（`/auth/signup`）にアクセスすると、ログインページにリダイレクトされます
- ✅ Cognito APIを直接呼び出しても、新規ユーザーの作成はできません
- ✅ 管理者のみがAWS Cognitoコンソールからユーザーを作成できます

## ユーザー作成

### Cognitoコンソールでのユーザー作成手順

1. **Cognitoコンソールにアクセス**
   ```
   AWS Management Console → Amazon Cognito → User pools
   ```

2. **User Poolを選択**
   - Amplifyが作成したUser Poolをクリック

3. **ユーザー作成画面を開く**
   ```
   上部の「Create user」ボタンをクリック
   ```

4. **ユーザー情報を入力**
   - **Invitation message**: 「Don't send an invitation」を選択（推奨）
   - **User name**: メールアドレスを入力（例: `user@example.com`）
   - **Email address**: 上記と同じメールアドレスを入力
   - **Email verified**: 「Mark email as verified」にチェック（重要）
   - **Temporary password**: 初期パスワードを設定
     - 自分で決める場合: 8文字以上、大文字・小文字・数字を含む
     - 自動生成する場合: 「Generate a password」を選択

5. **ユーザーの作成**
   - 「Create user」をクリック
   - 成功メッセージが表示されることを確認

6. **初回ログイン**
   - アプリのログイン画面（`/auth/signin`）にアクセス
   - 作成したメールアドレスとパスワードでログイン
   - 初回ログイン時にパスワード変更を求められる場合があります

### 推奨ユーザー設定

個人・家族使用の場合、以下のユーザーを作成することを推奨します:

- **自分用アカウント**: メインの管理者アカウント
- **家族用アカウント**: 配偶者や家族メンバー用（必要に応じて）

#### 例
```
ユーザー1: main-user@example.com （管理者）
ユーザー2: family-member@example.com （家族）
```

## Amazon SES設定（将来対応）

**現時点ではこの設定は不要です。Phase 2（メール自動取り込み機能）の実装時に必要になります。**

### 将来的な設定手順（参考）

1. **SESでメール受信の設定**
   - Amazon SESコンソールを開く
   - 受信ルールセットを作成
   - 専用メールアドレスを設定（例: `receipts@yourdomain.com`）

2. **S3バケットの設定**
   - メール保存用のS3バケットを作成
   - SESからのアクセス権限を付与

3. **Lambda関数の設定**
   - メール解析用のLambda関数をデプロイ
   - SESからLambdaへのトリガーを設定

**詳細は Phase 2 実装時に別途ドキュメント化します。**

## トラブルシューティング

### 1. デプロイに失敗する

**症状**: Amplifyのデプロイが「Failed」になる

**対処法**:
- ビルドログを確認する
- Node.jsのバージョンを確認（package.jsonで指定されているバージョン）
- 依存関係のインストールエラーがないか確認

### 2. ログインできない

**症状**: 正しいメールアドレスとパスワードを入力してもログインできない

**対処法**:
- Cognitoコンソールでユーザーが正しく作成されているか確認
- メールアドレスが検証済み（verified）になっているか確認
- パスワードが要件を満たしているか確認（8文字以上、大文字・小文字・数字を含む）
- ブラウザのコンソールログを確認してエラーメッセージを確認

### 3. サインアップページにアクセスできてしまう

**症状**: `/auth/signup` にアクセスしてもリダイレクトされない

**対処法**:
- Amplifyが最新のコードでデプロイされているか確認
- ブラウザのキャッシュをクリア
- シークレットモード（プライベートブラウジング）で試す

### 4. Cognito User Poolが見つからない

**症状**: CognitoコンソールにアクセスしてもUser Poolが表示されない

**対処法**:
- AWSリージョンを確認（Amplifyがデプロイされたリージョンと同じか）
- Amplifyのデプロイが完全に完了しているか確認
- IAMユーザーに適切な権限があるか確認

## セキュリティのベストプラクティス

### パスワードポリシー

- **最小文字数**: 8文字以上
- **複雑さ**: 大文字、小文字、数字を含む
- **定期的な変更**: 推奨（3〜6ヶ月ごと）

### アクセス管理

- **管理者アカウント**: AWS Cognitoへのアクセス権を持つユーザーを最小限に
- **MFA（多要素認証）**: AWSアカウントにはMFAを有効化することを強く推奨
- **アクセスログ**: AWS CloudTrailでCognitoの操作ログを記録

### データ保護

- **HTTPS**: 常にHTTPSでアプリにアクセス（Amplify Hostingは自動的にHTTPSを提供）
- **機密情報**: クレジットカード番号は保存しない（現在の設計では保存していません）
- **バックアップ**: 重要なデータは定期的にバックアップ
