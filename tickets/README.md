## 📝 開発の進め方

### 1. Phase 1 (MVP) を優先的に実装

まずは基本的な家計簿機能を完成させます。

**推奨実装順序**:
1. #01 ユーザー認証
2. #02 家族グループ管理
3. #05 カテゴリ管理（デフォルトカテゴリ作成を#02に統合）
4. #03 収支の手動入力
5. #04 収支一覧表示
6. #06 ルールベース自動分類

### 2. Phase 2 (自動化) で差別化

メール自動取り込み機能でユーザー体験を向上させます。

### 3. Phase 3 (レポート) で価値提供

分析機能で長期利用を促進します。

## ✅ チケットの進め方

### チケットを開始する前に
- [ ] チケットの内容を熟読
- [ ] 前提チケットが完了しているか確認
- [ ] 必要なツール・ライブラリをインストール

### 実装中
- [ ] 受入基準を1つずつ確認しながら実装
- [ ] コメントを添えてコードを記述（初心者でも分かるように）
- [ ] エラーハンドリングを適切に実装

### 実装完了時
- [ ] すべての受入基準を満たしているか確認
- [ ] テスト項目をすべて実行
- [ ] 完了条件をすべて満たしているか確認
- [ ] コミット & プッシュ
